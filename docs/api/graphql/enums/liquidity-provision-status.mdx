Status of a liquidity provision order


```graphql
enum LiquidityProvisionStatus {
  STATUS_ACTIVE
  STATUS_STOPPED
  STATUS_CANCELLED
  STATUS_REJECTED
  STATUS_UNDEPLOYED
  STATUS_PENDING
}
```


### Values

#### [<code style={{ fontWeight: 'normal' }}>LiquidityProvisionStatus.<b>STATUS_ACTIVE</b></code>](#) 
> An active liquidity provision
> 

#### [<code style={{ fontWeight: 'normal' }}>LiquidityProvisionStatus.<b>STATUS_STOPPED</b></code>](#) 
> A liquidity provision stopped by the network
> 

#### [<code style={{ fontWeight: 'normal' }}>LiquidityProvisionStatus.<b>STATUS_CANCELLED</b></code>](#) 
> A cancelled liquidity provision
> 

#### [<code style={{ fontWeight: 'normal' }}>LiquidityProvisionStatus.<b>STATUS_REJECTED</b></code>](#) 
> Liquidity provision was invalid and got rejected
> 

#### [<code style={{ fontWeight: 'normal' }}>LiquidityProvisionStatus.<b>STATUS_UNDEPLOYED</b></code>](#) 
> The liquidity provision is valid and accepted by the network, but orders aren't deployed
> 

#### [<code style={{ fontWeight: 'normal' }}>LiquidityProvisionStatus.<b>STATUS_PENDING</b></code>](#) 
> The liquidity provision is valid and accepted by the network, but orders aren't deployed and
> have never been deployed. If when it's possible to deploy them for the first time the
> margin check fails, then they will be cancelled without any penalties.
>
